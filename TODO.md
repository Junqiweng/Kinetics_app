# TODO / Roadmap

> 目标：保持“化工工程师友好”的简单流程，同时逐步增强数值稳健性与可解释性。

## 近期（高优先级）

- 增加“CSV 数据检查清单”按钮：一键检查缺列、非数字、单位范围（不启动拟合）
- 在 UI 中明确提示：所选测量列不允许 NaN/空值（并给出修复建议）
- 为 `k0`、`Ea`、`K0` 增加“单位/数量级提示”（避免初值差太多导致拟合困难）

## 中期（可选增强）

- 支持按实验误差提供权重（例如用户提供 `sigma_*` 列，或统一相对误差）
- 支持“只用部分行/工况拟合”（例如按温度、体积、时间筛选）
- 增加对“参数相关性”的简单提示（例如 $k_0$–$E_a$ 相关性）

## 长期（需要明确需求）

- 支持 CSTR（稳态）模型（与当前数据格式对齐）
- 增加能量方程/非等温（需要明确热效应与传热模型）
- 支持体积流量沿程变化、压降（更贴近气相/多相体系）

## 已知限制（当前版本）

- 仅支持 PFR 与 Batch
- PFR 假设恒定体积流量：$C_i=F_i/\\dot{v}$
- 所选测量列不能有 NaN/空值/非数字
